Chapter 8 Forms

What I learned from reading about Forms, is that forms are made up of a

element that contains form controls such as input fields, select menus and buttons. These input fields can be populated with information that is processed once the form has been submitted.

Traditionally, when a form was submitted, it would be sent to a server where the information would be processed using a 'back end' language such as PHP or Ruby. It’s possible, and becoming more and more common, to process the information in a form on the 'front end' before it is sent to the server using JavaScript, which is what we’ll be focusing on in this chapter.

Using Form Data Objects Effectively

In this video the author explains how you can get all elements in a Form element, without having to create variables for each one of them. To do it he uses the FormData object, which gets everything inside a form and then you can send that information to the server. Also, shows how you can turn your Form object into JSON, so that you can send JSON to the server instead. The most important things to remember from this video are, first, that all your inputs should have a name for the FormData object to work; second, you should prevent the submit button to send information by default using the “preventDefault()” method.

MDN: Client-site Form validation

From this article I learned that you can validate form information with the built-in validation attributes on HTML5, or using JS. JS is a better option if you need more specific form validation, and when browsers don’t support HTML5 form validation. There are two ways that you can implement form validation using JS, one is using the constraint validation API, and the other is using just plain JS, which is of course a little more complicated.
